Requirements

  • Plugin:

    • Browser: ie7+ , firefox 3.6+ , chrome , safari 5+, opera 11+
    • jQuery: 1.6+

  • Editor:

    • Just use a modern browser , note that you won't be able to use local image files in IE9 only URLs

Installation

Add jquery , upload plugin files from folder "Plugin/minified" , add the following to head tag "modify paths to where you uploaded the files"

        <link rel="stylesheet" type="text/css" href="jquery-hotspotter.min.css" />
        <!--[if lt IE 9]><link rel="stylesheet" type="text/css" href="ie7-8.min.css" /><![endif]-->
        
        <script type="text/javascript" src="jquery-hotspotter.min.js"></script>

        <script type="text/javascript">
        jQuery(document).ready(function($) {
            //Run plugin when DOM is ready
            $('.hs-area').hotspotter();
        });
        </script>

Done! , now you can insert code from the editor in your page

Note: the second line is only used to show Glass , Shadow spots in IE 7,8. if you don't use this spots or don't plan to support this browsers , remove this line

Quick Notes

- You can use arrow keys for more precise positioning of spots.

 

Using Images Inside Aiming Spot's Tooltip:
  • You must set width for tooltips that's shown on left and right , you can use 'Width' option on the editor or wrap your HTML in a 'div' and set it's width.
  • Also you must set height for tooltips shown on top or bottom by wrapping it's HTML content inside a 'div' & set it's height
  • Choose width or height that account for image dimension & other content in tooltip
  • Not setting the dimension in either case will break the tooltip & show the image outside it when the image loads

 

 

Spot Sharing:
  • If you use fixed top menu , you can add a margin between the image & top of browser window to prevent your top menu from overlapping with the image when user navigate to spot URL , to do so pass 'imgTopMargin' option when initializing the plugin , Example: 
                    jQuery('.hs-area').hotspotter({imgTopMargin: 50}); //defaults to 10px
  • This feature isn't support for IE7.

 

 

Responsive / Fluid Images:

If you checked the responsive option in the Editor , the plugin will detect image dimension, scale & reposition spots when the page load , but if you are going to resize the image with javascript 'like in demos' , or use fluid images you will need to rerun the plugin , the following code will rerun the plugin whenever the window resize so that you can use it with fluid images: 

jQuery(window).bind('resize', function() {
    setTimeout(function() {
        jQuery('.hs-area').hotspotter();
    }, 400);
});

Controlling spots with javascript

Show / hide all spots on image:

Showing:

jQuery('#area').hotspotter('show');

Hiding:

jQuery('#area').hotspotter('hide');

 

where '#area' refer to the container or the main div id that you have got from the editor , this div element has a class 'hs-area' , you will find it in the first line of the generated code

 

 

Trigger a spot:
jQuery('#myspot').hotspotter('trigger');


jQuery('#myspot').hotspotter('untrigger');

 

Get spot state active/inactive:

 

var isActive= jQuery('#myspot').hotspotter('isActive');

if (isActive) {
    //active
}else{
   //inactive
}

where '#spot' refer to spot div , the one with attribute 'data-coord'

Custom Spots

First you need to understand generated HTML:

Example HTML:

<div class="hs-area">
    <img data-imgdim="940,198" src="your image URL">
    <div class="hs-wrap">
        <div class="red-spot" data-name="test" data-activeon="hover" data-dim="76,47,10" data-coord="406.3,59"
        ></div>
        <div class="tt-wrap">
            <div class="tip-tooltip black" data-pos="-20" data-width="200" data-dir="top" data-anim="goin">
                Tooltip content
            </div>
        </div>
    </div>
</div>

As you can see it uses abbreviations for what you see in editor but with some alternations:

  • 'data-imgdim' is used to save original image dimensions , so when you use the same HTML above with larger or smaller version of the image the plugin will be able to scale & reposition spots to appear exactly in the same position.
  • Every spot / tooltip combination are grouped in a 'div' with class 'hs-wrap'
  • If you removed 'data-activeon' attribute only the spot will be shown and no tooltips or effects
  • 'data-dim' refer to spot dimension 'outerWidth, outerHeight, border-radius' in the same order
  • 'data-coord' refer to spot coordinates 'left, top' in px
  • Default spot types like 'red , glass, shadow , ..' are specified as class like above , see div with class 'red-spot'
  • same with tooltips you have 'tip-tooltip' , 'bubble-tooltip' , 'aim-tooltip'
  • the default for 'data-dir' is top so if you want tooltip direction to be top you can remove this attribute
  • "data-name" refer to hash link name , so you could access your spot with hash links like http://www.yoursitename.com/page.php#SpotName
  • The previous HTML indicate a 'general' spot to make it as 'aim' change code to
    <div class="hs-area">
    <img data-imgdim="940,198" src="your image URL">
    <div class="hs-wrap">
        <div class="red-spot" data-handler="Aim" data-name="test" data-activeon="hover" data-dim="76,47,10"
        data-coord="406.3,59" ></div>
        <div class="tt-wrap">
            <div class="aim-tooltip" data-dir="top" >
                <div class="tt-content">Tooltip content</div>
            </div>
        </div>
    </div>
</div>
    • we add 'data-handler=Aim' , we don't do the same with general spots because it's default
    • tooltip class became 'aim-tooltip'
    • we have remove 'data-pos' because it's non relevant with Aim tooltips it's always positioned relative to image
    • we have removed 'data-width' because the direction is top so the width is fixed with the image width , you can set width if you choose directions left or right
    • added additional div with class 'tt-content' to wrap tooltip content , this is useful to avoid the undesirable content reflow of tooltip with the animation , try put lot of content in tooltip , remove it & see difference
    • if you want to prevent aiming animation add attribute, data-handleropts= "false" , in spot div
  • finally if you want to use your spot as a link wrap it in an 'a' tag like 
    <div class="hs-area">
    <img data-imgdim="940,198" src="your image URL">
    <div class="hs-wrap">
        <a href="http://www.example.com">
            <div class="red-spot" data-name="test" data-activeon="hover" data-dim="76,47,10" data-coord="406.3,59"></div>
        </a>
        <div class="tt-wrap">
            <div class="tip-tooltip black" data-pos="-20" data-width="200" data-anim="goin" data-dir="top">Tooltip content</div>
    </div>
</div>
</div>

As you become acquainted with HTML structure you can now easily create custom spots.

Note that strings in 'data-handler' attribute are case sensitive , so don't write 'aim' instead use 'Aim'

 

 

Image spots:

This type of custom spot could easily be created using the editor , choose image spot from the left panel , click in the image where you want to insert it , a dialog will be opened to choose 2 images the first is the one shown by default , the second will be shown on hover/click. lets look at the markup level:

<div class="hs-area">
    <img data-imgdim="940,198" src="your image URL">
    <div class="hs-wrap">
        <div class="img-spot" data-activeon="hover" data-coord="407.3,43">
            <img src="default image URL">
            <img src="image on hover/click URL">
        </div>
        <div class="tt-wrap">
            <div class="tip-tooltip black" data-dir="top" data-anim="goin"></div>
        </div>
    </div>
</div>
  • we have used class 'img-spot'
  • added 2 'img' elements in spot , the first is the default , the second shown on hover/click

 

CSS spots:

Example code:

css

.my-spot { /*default spot style*/
   border: 4px solid #0078a3;
}
.my-spot.active { /*style applied when active hover/click*/
   border: 4px solid #000000;
}
.my-tooltip {
   border: 4px solid #cd0a0a;
}

HTML

<div class="hs-area">
    <img src="your image URL">
    <div class="hs-wrap">
        <div class="my-spot" data-activeon="hover" data-dim="49,26,0" data-coord="451,86"></div>
        <div class="tt-wrap">
            <div class="my-tooltip" data-dir="top" data-anim="goin" >Tooltip Content</div>
        </div>
    </div>
</div>

As you can see it's really simply you have made styles , let the plugin do the hard work for you , also your spot will be responsive.

HTML + CSS spots:

Example:

CSS: only use 'my-tooltip' class style from previous section

HTML:

<div class="hs-area">
    <img src="your image URL">
    <div class="hs-wrap">
        <div class="markup-spot" data-activeon="hover" data-dim="49,26,0" data-coord="451,86">
            <div>
                Default div
            </div>
            <div>
                hover/click div
            </div>
        </div>
        <div class="tt-wrap">
            <div class="my-tooltip" data-dir="top" data-anim="goin" >Tooltip Content</div>
        </div>
    </div>
</div>
  • This kind of spot contain 2 'div' elements the first will be shown by default , the second on hover/click
  • note that we used class 'markup-spot' , setting this class will allow putting the 2 divs , switching between them
  • As you can see you can do whatever you want with this div , add other element , add class , ...
  • we used the custom tooltip from previous section
  • Using this kind of spots your spot won't be responsive, unless your divs are fluid , you will not be able to use the responsive feature as the plugin only scale & reposition main spot div 'the one with data-coord attribute'
HTML + CSS + Javascript spots:

Still not satisfied with previous stuff and want to also use javascript , you can use the plugin as a framework that will loop through images & it's spots + tooltips , gather information , clear & bind events then use you own function! also your spots could still be triggered with 'hash' links , have a look at this example:

css

.my-spot {
   border: 4px solid #0078a3; /*default spot style*/
}
.my-spot.active {
   border: 4px solid #000000; /*style applied when active hover/click*/
}
.my-tooltip {
   border: 4px solid #cd0a0a;
}

 

HTML

<div class="hs-area">
    <img src="your image URL">
    <div class="hs-wrap">
        <div class="my-spot" data-handler="MyHandler" data-handleropts="op1,op2" data-activeon="hover" data-dim="49,26,0" data-coord="451,86" data-name="spotname"></div>
        <div class="tt-wrap">
            <div class="my-tooltip" data-dir="top" data-anim="goin" data-width='100' data-pos="-10">Tooltip Content</div>
        </div>
    </div>
</div>

 

Javascript

(function ($) {
   $.fn.hotspotter.MyHandler= {
      init: function() {
          //run with each spot with this type , only in initialization
      },
         
      enter: function() {
          //triggered with 'mouseenter' event with hover spots
      },
         
      leave: function() {
          //triggered with 'mouseleave' event with hover spots
      },
         
      click: function() {
          //triggered with 'click' event with hover spots
          //to avoid rewriting same code in enter/leave here, we call them!
          if (this.b_isActive) {
              this.enter();
          }else{
              this.leave();
            }
      },
         
      scale: function() {
          //This function is responsible for putting spot in the desired position in the image using data in 'this.coord' array
          //also setting dimension - if needed - using data in 'this.dim' array ,
          //if you made image responsive 'ie: save original image dimensions in 'data-imgdim'  attribute'
          //you will need to use data in 'this.generalData' to get change in image width & height & adjust spot
          //position & dimensions accordingly , look at table below to get more info about properties that's available to your handler
      },
         
      _myownfunction: function() {
          //you can add your own functions , it will be available in other object function in 'this._myownfunction'
          //as the plugin will get all object properties / methods not  only what's used directly by the plugin
      },
         
      _myownProperty: 5
  };
})(jQuery);

try the previous code & have a look at default handlers in plugin source code.

Data available to use in handler , accessed by 'this.optionName':

Option Name Refer to Value in Example
b_$wrap jquery object of the div with hs-wrap class  
b_$spot jquery object of spot div div with 'data-coord' attribute
b_name spot name test
b_coord spot coordinate supplied as array [left, top] 451,86
b_dim spot dimensions supplied as array [outerWidth, outerHeight, border-radius] 49,26,0
b_activeon 'data-activeon' attribute hover
b_hdOpts options supplied to your handler in 'data-handleropts', get it as array opt1, opt2
b_$tooltip jquery object of div with 'tt-wrap' class  
b_$inTooltip jquery object of tooltip div div with class 'my-tooltip'
b_tooltipW tooltip width , attribute 'data-width' 100
b_tooltipDir tooltip direction , attribute 'data-dir' top
b_tooltipPos tooltip position relative to element , attribute 'data-pos' -10
b_tooltipAnim tooltip animation , attribute 'data-anim' goin
b_isActive flag to know if spot is active or not , defaut to false , this flag is important to be able to use hash links , you must ensure that it's true when spot is active , false otherwise  
b_generalData

you can store general data shared between all image spots in this object.

It contain by default two properties 'dw' , 'dh' this properties is only used if the image is responsive , it will be populated with change in image width , height so:

dw = 'new image width' / 'original image width'

dh = 'new image height' / 'original image height'

Original image refer to the image you used in editor , original width & height of image are stored in attribute 'data-imgdim' in img dom element it's stored in the form 'data-imgdim="width, height"'

You can use those properties in your handler 'scale' function to make your spot responsive.

  

 

  • Note that handler name is case sensitive so whatever the name you choose for your plugin , you must use it exactly in "data-handler" attribute.
  • Options starting with $ refer to the corresponding jquery object of the HTML element.
  • You will get 'b_coord' , 'b_dim' as arrays with the same value in the corresponding attribute with 'parseFloat' function applied.
  • Get parseFloat of 'b_tooltipPos'.
  • If you choose the names 'General' or 'Aim' for your handler , your handler will overwrite the default ones
  • You will need to change 'isActive' state with 'enter' & 'leave' to be able to use the 'click' function provided above with spots that require click , also to allow triggering spot with hash links.
